4 出现的问题

在终端中执行 import mujoco_py 时出现如下问题：

  1. fata error: X11/Xlib.h: No such file or directory
  2. fatal error: GL/glew.h: No such file or directory
  3. No such file or directory: ‘patchelf’

解决办法： 安装第3.2节的依赖。

终端能正常运行后，运行示例程序时直接卡死，终端提示 ERROR: GLEW initalization error: Missing GL version。

解决办法： 在 ~/.bashrc 中添加 export LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libGLEW.so

5 Pycharm 中运行mujoco_py会出现的问题

在Pycharm中运行mujoco_py会出现环境变量的问题，解决方法是手动添加环境变量。
Pycharm -> Run -> Edit Configurations -> Environment Variables，添加：
$LD_LIBRARY_PATH:/home/robot/.mujoco/mujoco210/bin:$LD_LIBRARY_PATH:/usr/lib/nvidia

注意两个环境变量中间用一个:隔开

再添加用于渲染的环境变量，变量名LD_PRELOAD，变量值/usr/lib/x86_64-linux-gnu/libGLEW.so

6 Windows下的安装

参考：https://zhuanlan.zhihu.com/p/462948069

6.1 流程及安装包下载

在Windows上装mujoco_py的流程分为三步：

  1. 安装Microsoft Visual C++ 2015，安装这个东西主要为了解决两个顽疾：
    • Microsoft Visual C++ 14.0 or later is required
    • distutils.errors.CompileError: command 'C:\\Program Files\\Microsoft Visual Studio\\2022\\Community\\VC\\Tools\\MSVC\\14.30.30705\\bin\\HostX86\\x64\\cl.exe' failed with exit code 2
      这两个顽疾是因为Mujoco底层是用C语言写的，所以需要这个编译器。但是新版本的安装很麻烦，又要选择组件，又要选择C++，一个不慎就是白干，而且新版本的安装会很大，不值得。因此就用2015版本的，其实正好对应的是提示里面的14.0版本。
      VS2015 下载地址：百度网盘：https://pan.baidu.com/s/1Nh7cCrqyroEMH90HUYbvfw 提取码：5251
      官网上也可以下载，找起来比较麻烦。下载完后点击名字带有full的那个安装包，自定义安装全部勾选上即可。
  2. mujoco210-windows：https://github.com/deepmind/mujoco/releases/download/2.1.0/mujoco210-windows-x86_64.zip. 下载下来解压至C:\Users\YourName\.mujoco目录。
  3. 环境变量->系统变量->Path 新建两条：
    C:\Users\YourName\.mujoco\mujoco210\bin
    C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\bin

6.2 报错及解决

ImportError: DLL load failed while importing cymj: The specified module could not be found..
参考DLL错误：https://github.com/openai/mujoco-py/issues/638。
解决方案：在import os后添加os.add_dll_directory("....... //.mujoco//mjpro150//bin")。

另外就是6.1节提到的C++编译器的问题，建议卸载掉其他版本的VS相关的所有内容，单独装百度网盘里的那个2015版本的.
